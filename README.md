# NixOS configuration for our family

Multi-host NixOS configuration for Matthew's machines.

This is a flake-based config managing several hosts with shared modules.
Currently active: `sloane` (workstation). Forthcoming: `isard` (legacy
hardware), `mayfeld` (macOS via nix-darwin / home-manager), `war-machine`,
`general-organa`, `gonk` (homelab).

## Bootstrapping a new install

These instructions are for a fresh install on `sloane`. Adapt for other
hosts as they get added.

### Prerequisites

- NixOS minimal install ISO on a USB drive (download from
  [nixos.org/download](https://nixos.org/download))
- Network connectivity (Ethernet preferred for install speed)
- Any data on the existing disk backed up — the install destroys it

### 1. Boot the live ISO

Boot the machine from the USB. At the live shell prompt:

```bash
# Become root
sudo -i

# Connect to wifi if needed (skip for ethernet)
nmtui  # or nmcli device wifi connect SSID password PASSWORD

# Verify connectivity
ping -c 3 nixos.org
```

### 2. Partition the disks

`sloane` has two NVMe drives:

- `nvme0n1` — for root (will be wiped, btrfs + LUKS)
- `nvme1n1` — `/games`, ext4, **PRESERVE** (don't touch)

Identify the disks:

```bash
lsblk
```

The disk with `/games` should be visible by label. Confirm before proceeding.
Don't format or partition that disk.

Partition the root disk. Using `cfdisk` for clarity:

```bash
cfdisk /dev/nvme0n1
```

Create:
- `/dev/nvme0n1p1` — 1 GiB, type "EFI System" (ESP)
- `/dev/nvme0n1p2` — remaining space, type "Linux filesystem" (will be LUKS)

Write changes, exit `cfdisk`.

### 3. Format

Format the ESP:

```bash
mkfs.fat -F 32 -n EFI /dev/nvme0n1p1
```

Set up LUKS on the root partition:

```bash
cryptsetup luksFormat /dev/nvme0n1p2
# Type "YES" when prompted, then choose a strong passphrase
cryptsetup open /dev/nvme0n1p2 cryptroot
```

Format the unlocked container as btrfs:

```bash
mkfs.btrfs -L nixos /dev/mapper/cryptroot
```

### 4. Create btrfs subvolumes

Mount the btrfs and create subvolumes:

```bash
mount /dev/mapper/cryptroot /mnt
btrfs subvolume create /mnt/@
btrfs subvolume create /mnt/@home
btrfs subvolume create /mnt/@nix
btrfs subvolume create /mnt/@swap
btrfs subvolume create /mnt/@snapshots
umount /mnt
```

### 5. Mount everything

Mount the subvolumes with appropriate options. `compress=zstd` enables
transparent compression; `noatime` reduces unnecessary writes; `ssd` is
auto-detected but explicit is fine.

```bash
# Mount root subvolume
mount -o subvol=@,compress=zstd,noatime,ssd /dev/mapper/cryptroot /mnt

# Create mountpoints
mkdir -p /mnt/{home,nix,swap,.snapshots,boot,games}

# Mount the other subvolumes
mount -o subvol=@home,compress=zstd,noatime,ssd       /dev/mapper/cryptroot /mnt/home
mount -o subvol=@nix,compress=zstd,noatime,ssd        /dev/mapper/cryptroot /mnt/nix
mount -o subvol=@snapshots,compress=zstd,noatime,ssd  /dev/mapper/cryptroot /mnt/.snapshots

# Mount the @swap subvolume separately (no compression for swapfile)
mount -o subvol=@swap,noatime,ssd                     /dev/mapper/cryptroot /mnt/swap

# Mount the ESP
mount /dev/nvme0n1p1 /mnt/boot

# Mount the existing /games (preserve from previous install)
mount /dev/nvme1n1p1 /mnt/games  # adjust device path if different
```

### 6. Create swapfile

8GB swapfile as an OOM safety net. Pairs with zram (configured in
the workstation profile) which handles primary memory-pressure relief
in compressed RAM. We don't use this for hibernation — it's intentionally
small.

```bash
btrfs filesystem mkswapfile --size 8g --uuid clear /mnt/swap/swapfile
swapon /mnt/swap/swapfile
```

### 7. Generate hardware-configuration

```bash
nixos-generate-config --root /mnt
```

This writes `/mnt/etc/nixos/configuration.nix` and
`/mnt/etc/nixos/hardware-configuration.nix`. We only need the latter.

### 8. Clone this flake

```bash
nix-shell -p git
git clone https://github.com/llama04108/nixos-config /mnt/etc/nixos-flake
```

(Adjust the URL if you've forked or moved the repo.)

### 9. Replace placeholder hardware-configuration

```bash
cp /mnt/etc/nixos/hardware-configuration.nix \
   /mnt/etc/nixos-flake/hosts/sloane/hardware-configuration.nix
```

Verify the generated file declares the expected paths (LUKS device, btrfs
subvolumes, fileSystems entries):

```bash
cat /mnt/etc/nixos-flake/hosts/sloane/hardware-configuration.nix
```

If something looks wrong, edit by hand.

### 10. Install

```bash
nixos-install --flake /mnt/etc/nixos-flake#sloane
```

This builds the entire system inside the chroot, sets up the bootloader,
and prepares for first boot.

When prompted for the root password, set one (or skip with empty password
since we're not using root login over SSH).

### 11. Move the flake to its final location

After the install completes successfully, before reboot:

```bash
mkdir -p /mnt/home/matthew/code
cp -r /mnt/etc/nixos-flake /mnt/home/matthew/code/nixos-config
chown -R 1000:100 /mnt/home/matthew/code/nixos-config
# (UID/GID 1000:100 should be matthew:users; adjust if needed)
```

This is so you can `nixos-rebuild` from your home directory after boot.

### 12. Reboot

```bash
reboot
```

Remove the USB. The machine should boot into NixOS, prompt for the LUKS
passphrase, then show the SDDM login screen.

### 13. First login

Log in as `matthew` with password `changeme`. Then proceed to
[POST_INSTALL.md](./POST_INSTALL.md) for the imperative steps that
have to happen after the rebuild.

## Day-to-day operations

### Rebuild after a config change

```bash
cd ~/code/nixos-config
sudo nixos-rebuild switch --flake .#sloane
```

Aliases configured in fish (via `users/matthew/default.nix`):
- `nrs` → `sudo nixos-rebuild switch --flake .`
- `nrt` → `sudo nixos-rebuild test --flake .`  (try without persisting)
- `nrb` → `sudo nixos-rebuild build --flake .` (build only, don't activate)

### Update the flake

Updates pinned input versions in `flake.lock`:

```bash
nix flake update
```

Or update a specific input:

```bash
nix flake update nixpkgs
```

Then rebuild to apply.

### Validate without applying

```bash
nix flake check
nix build .#nixosConfigurations.sloane.config.system.build.toplevel --dry-run
```

Useful for catching errors before `nixos-rebuild switch`.

### Roll back

NixOS keeps previous generations of the system. To boot a previous
generation, select it from the systemd-boot menu at boot. Or to
explicitly roll back to the last working generation:

```bash
sudo nixos-rebuild --rollback switch
```

To list generations:

```bash
sudo nix-env --list-generations --profile /nix/var/nix/profiles/system
```

### Theme experimentation

Edit `modules/nixos/desktop/theming.nix` and change the `theme` variable
in the `let` block. Available schemes: see comments in that file.
Rebuild. Most surfaces (~90%) update automatically. Noctalia, Brave, and
Steam need separate config changes if you want them to follow.

## Adding a new host

1. Create `hosts/<hostname>/default.nix` and
   `hosts/<hostname>/hardware-configuration.nix`
2. Add the host to `flake.nix`'s `nixosConfigurations`:
```nix
   nixosConfigurations.<hostname> = mkHost "<hostname>" [ ];
```
3. Reference appropriate profile(s) in the host's `default.nix`
4. Test with `nix flake check` and `nix build`
5. Install via the bootstrap steps above, substituting the hostname
